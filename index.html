0x1000000 = 0xffffff + 1保证了选取范围，与0进行位运算，0x000000~0xffffff范围内的所有整数值都有可能得到了，不会漏值，拼接 ‘00000’，切片-6，保证了值的长度为6位，不会出现颜色值偏差。’#38F’跟 ‘#00038F’颜色值肯定是不同的。

目前来看，我认为，这应该是最nice的随机颜色值生成方案了。

下面谈谈一些JavaScript数组的相关概念：

稀疏数组

稀疏就是不稠密，顾名思义，稀疏数组就是索引不连续的数组。

var a = new Array(5);    //a为空，但a.length=5
var b = [];              //b.length=0
b[10000] = 1;            //此时b只有一个元素，但b.length=10001

通常来说，数组的length属性也即数组中元素的个数，如果数组是稀疏的，那么，length的值将大于元素的个数。出现这种情况也是跟JavaScript中数组length属性的特殊性息息相关的。所以，可以通过以上两种方法来创建稀疏数组，但是，实际应用中，遇到创建稀疏数组的情况少之又少。

注意：当使用delete操作符来删除数组中元素，或者在数组直接量的声明中存在省略值时，也会造成稀疏数组的出现。

var a = [1, 2, 3];
var b = [1,,,];          //b.length=3
delete a[0];            //[undefined, 2, 3], a.length = 3

这是因为，

  • 用delete删除数组元素时，并不会改变length的大小，而且，也不会去移动索引值，所以，删除过元素，便会出现length大于数组元素的情况。
  • 省略元素声明的话，则会默认赋予undefined，一样会出现稀疏。

最后要说明的就是，当元素是undefined或空时，虽然有微妙的差别，但都是稀疏的表现。

数组类型检测

因为数组是一种特殊的Object类型，重载了许多有用的方法和属性，而且由于数组有序序列的特殊性，在JavaScript中，同等操作数组相比对象要快许多。所以，给出一个对象类型，来检测其是否为数组，通常还是蛮有用的。

JavaScript语言核心学习（类型，值和变量）

接触前端有一段时间了，却一直也没有花整块儿的时间对JS核心做深入学习。所以这次打算把犀牛书彻底啃下来。对一些核心知识点做个系统总结，还能顺带提升下写blog的软技能。

本文主要整理了JS中的类型，值，变量相关问题。废话不多说，直接上图。

类型

JavaScript的数据类型分为两类：原始类型(Primitive Type)和对象类型(Object Type)。

原始类型也即所谓原始值。它们是不能通过动态改变的。
而这里的对象类型实际上是广义的对象值。
因为JavaScript是动态弱类型的语言，所以，一些传统复杂类型在JavaScript中都被当作对象来处理。

另外还有两种分类方式：

拥有方法的类型和不能拥有方法的类型。从这个角度看的话，只有null和undefined被分出来，因为它们无法拥有方法。

可变类型和不可变类型。从这个角度看的话，5类原始值便是不可变类型。因为它们是不可被修改的。这里提一下字符串，虽然可以访问字符串任意位置的文本，但JavaScript本身并不提供修改已知字符串文本内容的方法。而所有的对象类型均是可动态被修改的。

Number

数字类型的话，需要注意的点是：不同于传统静态语言，JavaScript中的所有数字均用64位浮点数值表示。但是，当做实际操作时，则是基于32位整数，这个转换对用户来讲是透明的。因为这里是概括性总结，详细的一些小问题，我会在后期的文章中整理。

String

字符串是一组由16位值组成的不可变的有序序列。每个字符通常来自Unicode字符集。在EC5 中字符串是可以当作只读数组来处理的。

Boolean

布尔值想来是再熟悉不过了。但是，在JavaScript中，要留意类型转换这一点。对布尔值的影响还是蛮大的。

undefined/null

之所以把它们两个放在一起，是因为可以抽象把这两个原始值都理解为空。或者说，可以说undefined是空的原始值变量，null是空的对象类型变量。但是还是有很微妙的不同的。关于它们也有很多小细节。这里暂时不一一说了。

包装对象

包装对象可以说是JavaScript特立独行的一个概念。理解它便是理解了操作原始值的过程。我们知道字符串也有属性和方法：

var s = 'hello';
console.log(s.length);    //==5,为什么会有属性？
s.bar = 4;               //设置一个属性，并不会报错
var t = s.bar;          //undefined

可是为什么呢？
其实执行过程是这样的：当引用字符串s的属性时，JavaScript就会将字符串通过调用 new String(s)方式转换为对象，这个对象是继承字符串的所有方法和属性的。但是这只是个临时对象，一旦引用结束，这个刚刚创建的对象也就销毁了。所以，给s添加属性，并不会抱错，但是等到下一次你再想访问时，那个你添过属性或方法的对象早已不存在了。当然也就访问不到了。也即：修改只是发生在临时对象身上，而临时对象并没有保存下来。

这个临时对象，便是所谓的包装对象。一般来讲，包装对象的过程是一个实现细节，是隐式的。你也可以通过Number(),String(),Boolean()构造函数来显式创建包装对象。但好像实际应用中意义并不是很大。但这对理解JavaScript万事皆对象有一定帮助。

变量

这里想说一下命名提升。因为这发生在变量的声明过程。

var a = 'global';
function foo() {
    console.log(a);    //==undefined
    var a = 'local';      
    console.log(a);    //=='local'
}

输出结果还是比较奇葩的。实际上，以上代码等价于：

var a = 'global';
function foo() {
    var a;
    console.log(a);    //==undefined
    a = 'local';      
    console.log(a);    //=='local'
}

这样看起来就明晰许多了。
原理是：JavaScript函数里声明的所有变量（不涉及赋值），都被“提升”至当前作用域的顶部，本例便是foo函数。
所以，这里就有一个JavsScript的最佳实践在了：可以特意将变量声明放在作用域的顶部，以免出现不如人意的bug。

这个例子涉及到了一点作用域的理解。我会在下次整理出变量作用域和作用域链的相关理解。但是不伤大雅。

div居中的几种方式

在固定布局的页面之中，元素居中可谓是一项经常要用到的技能，刚开始的时候做一些简单页面也总是迷惑。这次进行下学习和总结。
总的来说，CSS来操作元素居中主要有四种方式。接下来一一介绍。

页面结构啥的我也就不写了，总之，不管是全局还是局部，#container都是要居中元素的外部容器。

1、使用自动外边距实现居中

这里要注意的就是：容器是必须指定宽度的，而且这种方式也被认为是最合理，最正确的方式，由于左右的自动外边距，可以实现窗口的自适应。

2、使用text-align实现居中

首先这是一个hack，因为text-align是文本属性，而这里却把它用在了元素定位上。这就会导致body的所有子元素都会居中显示，所以我们还需要额外的p {text-align: left;}来使文本内容恢复默认居左。另外，真正遵循标准的浏览器，并不会移动元素，而只是将文本居中。
实测：chrome, FF, SF, OP, IE8以上都不支持居中 ，只有IE6可以实现居中效果。*/

3、组合使用自动外边距和文本对齐

这其实就是前两种方式的结合，好处就是保证了跨浏览器的支持，但是仍需要写附加的左对齐规则，不甚完美。

4、负外边距解决方案

这最后一种方案使用了绝对定位+负外边距的解决，没有hack，适用性极广，而且最广的浏览器支持。

综上所述：
4种方案各有所长，如果想更多地跨浏览器支持，第四或第三种方式较好。更推荐第四。
若是现代浏览器，则毫无疑问使用第一种方案。

bower retry 的问题解决


感觉每天都有新的JavaScript库，框架或者衍生工具发布，让人应接不暇。而自己连目前已经成熟或者有了充分应用的那些还都没有很好地加以学习。仍处在蹒跚学步状态，实在对自己感到捉急。

这次遇到了一个关于Bower的问题。Bower是一个基于Node的包管理器。使用起来非常方便。

目前在学习AngularJs的过程中，发现其官方网站http://angularjs.org和谐严重。又想要拿到最新源码，小费了一翻周折。PS：方法有许多。只是写下自己解决问题的过程。

其实很简单，正常情况下bower install angular即可解决问题。但出现：

$ bower install angular
bower retry Request to https://bower.herokuapp.com/packages/angular failed with ETIMEDOUT, retrying in 1.1s

问题最后发现是代理设置的原因。解决如下：
在项目根目录下新建 .bowerrc 文件。如果已有直接修改即可。

{
    "directory": "public/components",
    "registry": "http://bower.herokuapp.com",
    "proxy": "http://<USERNAME>:<PASSWORD>@<PROXY_IP>:<PROXY_PORT>",
    "https-proxy": "http://<USERNAME>:<PASSWORD>@<PROXY_IP>:<PROXY_PORT>"
}

由于bower是从github进行源码下载。所以，如果本机有代理，系统代理的环境变量和git的.gitconfig也要预先设置。

最后问题完美解决。而且也认清到一点：
使用命令行安装远程文件，当出现 Request ETIMEOUT 时，首先检查当前的网络连接，如果没有问题。那么很大的可能性是当前的网络代理设置问题造成了访问超时。
