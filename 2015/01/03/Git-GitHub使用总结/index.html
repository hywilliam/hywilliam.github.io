fork的项目副本怎么与源项目的更新保持同步

如题，我们在开发过程中为了保证不污染源项目，通常fork一份远程副本然后再clone到自己本地进行开发修改。而后再适时push到自己的远端，继而pull request到源项目等待review。但是，经常有可能源项目进行了更新，而自己的远程副本没法实时同步，导致版本冲突。

解决问题需要明白此处有三个版本库的关系。远程源项目，自己的源项目副本，本地版本库。

最后，总结下来，当github上fork的副本和源项目不一致时的解决方案：

  1. git remote add upstream [源url] —— 增添源项目本地关联
  2. git fetch upstream —— 获取源项目更新
  3. git merge upstream/master —— 合并源项目到本地
  4. git push origin master —— push本地repo到自己的fork副本
